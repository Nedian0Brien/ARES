3Late Interaction

Single-vector retrievers encode a query and a passage into one dense embedding each and score them by a single dot product. This is fast, but it forces the model to compress every token interaction into one point in space. Late interaction instead retains a per-token embedding for the query and the document, and defers their comparison to scoring time. Each query token is matched to its most similar document token, and the resulting maximum similarities are summed [18].

(1)

Here Eq and Ed are the token-level embedding matrices. The MaxSim operator in Eq. (1) is cheap because document embeddings are pre-computed and indexed; only the per-query maxima are evaluated at search time.

3.1Denoised Supervision

ColBERTv2 improves the original model along two axes. First, it distills late-interaction scores from a stronger cross-encoder teacher, which sharpens the token alignments. Second, it compresses the index with residual quantization, storing each embedding as a centroid index plus a low-bit residual. The two changes are complementary: distillation raises quality while residual compression keeps the per-token index small enough to be practical.

Figure 3. MRR@10 on the BEIR out-of-domain suite. Late interaction (ColBERTv2) recovers fine-grained matches that single-vector DPR averages away, without the index cost of full cross-encoders.

3.2Why It Helps

The benefit is most visible on queries with rare entities or multi-term constraints, where a single averaged vector dilutes the decisive token. MaxSim preserves that signal because the contributing token is scored directly [22]. The cost is a larger index; §4 shows residual compression cuts it by an order of magnitude with negligible quality loss.

Why does late interaction beat single-vector retrieval here, and what does it cost?

It keeps per-token signal that a single vector averages away. Single-vector DPR compresses a passage into one embedding, so a decisive rare token gets diluted. Late interaction scores every query token against its best-matching document token with MaxSim, so that token is scored directly p4·S1.

The cost is index size — one embedding per token. ColBERTv2 makes it practical with residual compression, cutting the index ~10× at negligible quality loss p4·S2, which is why it stays ahead of SPLADE on BEIR Wiki.

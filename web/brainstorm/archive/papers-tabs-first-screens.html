Library / Lab / Wiki / Agent 역할과 첫 화면

A안의 핵심은 단순한 라벨 교체가 아닙니다. 각 탭은 논문을 읽는 동안 생기는 다음 행동을 맡고, 첫 화면은 사용자가 곧바로 읽고 확인하고 저장할 수 있어야 합니다.

public board · ares.lawdigest.kr

탭 정의 원칙

  1. 탭은 기능 묶음이 아니라 사용자의 다음 행동을 맡는다.
  2. 첫 화면은 빈 설명이 아니라 실제 작업 가능한 상태로 시작한다.
  3. 논문과 source link가 끊기면 탭 전환도 실패로 본다.
  4. 모바일에서는 탭마다 하나의 주 행동만 전면에 둔다.

상태 이동

  • Library: paper 선택, 읽기, 하이라이트
  • Lab: 비교 질문, 재현 계획, 결과 확인
  • Wiki: 개념 탐색, 연결 확인, 문서 열기
  • Agent: 질문, 근거 확인, 답변 저장

Library

논문을 고르고 읽는 기본 작업대입니다. 검색보다 현재 읽을 논문과 이어 읽기를 먼저 둡니다.

첫 화면
저장된 논문 목록 + 선택 논문 Reader preview
기본 액션
Open PDF, View text, Highlight, New note
저장 단위
Paper, Highlight, Note, Summary card

Lab

읽기에서 나온 의문을 검증 작업으로 바꿉니다. 재현 실험은 여기 남기되, 출발점은 paper context입니다.

첫 화면
현재 논문의 검증 질문, 비교 대상, 진행 중인 run
기본 액션
Create check, Compare papers, Attach result
저장 단위
Comparison, Reproduction plan, Result dossier, Idea

Wiki

읽은 논문들의 지식 지도입니다. 개별 요약 보관함이 아니라 논문 사이의 개념 연결을 보여줍니다.

첫 화면
현재 논문 중심 그래프 + 관련 concept/document list
기본 액션
Open document, Link note, Create synthesis
저장 단위
Wiki document, Entity, Concept, Synthesis

Agent

검색창이 아니라 리딩 파트너입니다. 답변보다 근거 확인과 저장 가능한 산출물이 중요합니다.

첫 화면
현재 paper context + source basket + 질문 thread
기본 액션
Ask, Cite source, Save as Note, Save as Idea
저장 단위
Agent response, Summary card, Note, Idea

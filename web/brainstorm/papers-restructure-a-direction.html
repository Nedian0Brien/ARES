Direction A

ARES는 유지하고, 첫 경험은 Papers로 바꾼다

ARES의 실험·분석 자산은 버리지 않습니다. 다만 사용자가 들어왔을 때 가장 먼저 만나는 것은 논문 목록과 리더입니다. Lab, Wiki, Agent는 논문 읽기를 도와주는 축으로 재배치합니다.

공개 확인 https://ares.lawdigest.kr/brainstorm/papers-restructure-a-direction.html

결정

제품명은 ARES로 둔다. 제품의 첫 목표는 논문 리딩이고, Papers는 그 리딩 경험을 설명하는 제품 개념으로 쓴다.

바뀌는 점

  1. 첫 화면은 논문 라이브러리와 Reader 중심으로 시작한다.
  2. Insight와 Writing은 독립 탭에서 빠지고 Wiki와 Agent 안의 산출물 흐름으로 흡수한다.
  3. Lab은 재현 실험보다 읽기에서 나온 비교·검증 작업을 먼저 받는다.
  4. Agent 답변은 저장 가능한 카드, 노트, 아이디어로 이어진다.

지켜야 할 것

  • PDF와 본문이 항상 화면의 중심이어야 한다.
  • 근거는 chip 장식이 아니라 이동 가능한 source로 보여준다.
  • 저장되지 않는 액션은 버튼처럼 보이지 않게 한다.
  • 모바일에서도 읽기, 질문, 저장은 끊기지 않아야 한다.
Library

논문을 고르고 읽는 기본 화면. PDF, 추출 텍스트, 요약, HTML 문서, 하이라이트가 여기서 열린다.

Lab

비교 읽기, 재현 계획, 실험 결과를 다룬다. Reader에서 넘어온 근거를 잃지 않는다.

Wiki

개념, 방법, 벤치마크, 실패 모드를 논문들과 연결한다. 그래프·리스트·그리드로 전환한다.

Agent

현재 논문과 Wiki, note를 보고 답한다. 답변은 Summary card, Note, Idea로 저장된다.

Adaptive Retrieval for Reasoning

PDF view · p.4 · Method

Retrieval Control

The system decides whether another retrieval step is needed before producing the next reasoning state. The core assumption is that uncertainty can be read from the current evidence set.

When evidence is partial, the model should ask for another source instead of completing the claim. This is the point the reader is likely to compare with other agentic verification papers.

기존 구조 매핑

현재 재편 후 이유
Read Library 검색보다 읽는 논문 목록과 Reader를 먼저 보여준다.
Lab Lab 기존 재현·비교 자산을 유지하되 Reader handoff를 더 강하게 묶는다.
Insight Wiki 개별 카드보다 논문 사이의 개념 연결을 우선한다.
Write Agent 초안 작성은 독립 탭보다 Agent 답변 저장과 산출물 흐름 안에 둔다.

기존 route alias는 바로 끊지 않는다. 내부 stage는 호환성을 유지하고, 상위 탭 라벨과 첫 화면을 먼저 바꾼다.

읽는 동안 생기는 산출물

Summary card

논문의 핵심 내용을 짧게 남긴다. Agent 답변에서도 만들 수 있다.

Highlight

PDF나 텍스트의 원문 위치와 함께 저장한다.

Note

사용자의 해석, 질문, 비교를 논문과 source에 붙인다.

Idea

후속 실험, 연구 질문, 제품 아이디어를 Lab이나 Wiki로 보낸다.

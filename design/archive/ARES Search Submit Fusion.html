검색창 Submit Fusion · 3 Variants

좌측 Agent / Keyword 모드 토글과 우측 Search 버튼을 하나의 컨트롤로 합치는 세 가지 디자인. 각 샘플은 실제로 동작합니다 — 비활성 모드를 클릭하면 모드가 전환되고, 활성 모드(= Search 버튼)를 클릭하면 검색이 실행됩니다.

Variant A · Split Pill Segmented submit

단일 pill을 두 칸으로 나눈다. 활성 쪽은 테마 컬러로 채워지고 "Search" CTA 역할을 겸한다. 비활성 쪽은 그림자 없이 연한 배경으로 남아 모드 토글 기능만 한다. 한 번의 클릭으로 모드 전환 + 즉시 검색까지 이어질 수 있어 동선이 가장 짧다.

활성 탭을 다시 클릭하면 검색 실행. 비활성 탭 클릭 시 모드 전환 + 자동 submit.
장점 · 가장 compact, 한 번의 클릭으로 모드+검색 단점 · "모드 전환"과 "검색"이 같은 UI라 의도 분리가 다소 모호 적합 · 데스크톱 파워 유저 / 숙련자 동선

Variant B · Twin Action Pills Dual buttons

두 개의 분리된 pill 버튼이 나란히 선다. 활성 모드 버튼은 확장되어 아이콘 + "Search" 라벨을 보여주고, 비활성 모드는 아이콘만 남은 동그란 칩으로 축소된다. 전환 시 두 버튼이 부드럽게 width morph하며 자리를 바꿔서 상태 변화가 시각적으로 명확하다.

활성 버튼 클릭 → 검색 실행. 비활성(아이콘) 클릭 → 모드 전환.
장점 · 두 모드가 물리적으로 항상 보여 발견성 높음 단점 · Variant A보다 가로 공간 더 차지 적합 · 모바일/신규 사용자 — 아이콘으로도 모드 의미 전달

Variant C · Morphing Primary Single CTA + swap

버튼은 하나다. 라벨과 색이 모드에 따라 "Agent Search" ↔ "Keyword Search"로 모핑한다. 왼쪽에 붙은 작은 원형 스왑 칩이 모드 전환 담당. "검색 버튼은 하나"라는 정신 모델을 유지하면서 모드를 부가 액션으로 분리하는 접근.

⇄ 아이콘 클릭 → 모드 스왑. 본체 버튼 클릭 → 검색 실행.
장점 · 단일 primary CTA로 의도가 가장 명확 단점 · 비활성 모드가 UI에 안 보여 모드 존재감이 약함 적합 · "검색하러 왔다"는 목적이 또렷한 홈/focal 화면

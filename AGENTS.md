# 🤖 Agent Work Instructions

## 🔄 1. 작업 프로세스 (Workflow)
- **사전 검토**: 작업 시작 전 `design/ARES Design System.html`을 분석하여 요구사항과 일치하는 디자인 토큰 및 컴포넌트를 확인합니다.
- **코드 품질**: 작업 완료 후 Lint 검사 및 빌드 테스트를 수행하여 오류가 없는지 확인합니다.
- **자동화**: 모든 작업이 정상 완료되면 다음 규칙에 따라 커밋 및 푸시를 진행합니다.
  - 커밋 메시지 형식: `feat: <내용>`, `fix: <내용>`, `style: <내용>` 등 
  - 메시지는 한국어로 작성하며, 변경 사항을 명확히 요약합니다.

## 🎨 2. 디자인 및 CSS 지침
- **ARES 디자인 시스템 준수**: 모든 UI 요소는 지정된 가이드를 따르며, 임의의 인라인 스타일링을 지양합니다.
- **토큰 기반 스타일링**:
  - 색상, 여백, 폰트 크기는 반드시 공통 변수(CSS Variables 등)를 사용합니다.
  - 라이트/다크 테마 전환 시 자연스러운 색상 변이가 이루어지도록 시스템 토큰을 매핑합니다.
- **CSS 최적화**:
  - 중복되는 스타일은 공통 클래스로 모듈화하여 파일 용량 증가를 억제합니다.
  - 사용하지 않는 클래스나 구형 브라우저용 프리픽스는 최소화합니다.

## ⚠️ 3. 주의 사항
- 기존 코드의 로직을 변경할 때는 반드시 하위 호환성을 고려합니다.
- 디자인 시스템에 정의되지 않은 예외 케이스 발생 시, 최대한 시스템과 유사한 톤앤매너로 제안합니다.
